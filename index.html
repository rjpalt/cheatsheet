Cheatsheet for Remembering All Those Semantic Elements Out There

Why go for semantic HTML?

Semantic HTML (and CSS) seem to be the thing of the future, but a lot of material out there still uses outdated semantics for their HTML and CSS. Lots of Teach-Yourself-HTML&CSS -courses use examples and excercises that are inconsistent with the so called best practices of semantic HTML.


Given the inconsistent quality of examples and exercises it is all too easy to fall back to bad old habits (suc as div soup). In order to better remember the semantic HTML elements and help me stay on the straight and narrow I've gathered some semantic elements into a table to better keep them in mind and readily available. Later on a mind map for semantic elements could be a nice idea. I've also collected some good resources at the end of the page for further reference.
